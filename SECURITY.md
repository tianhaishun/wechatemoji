# 安全说明

感谢你帮助 `WeChatEmoji` 变得更安全。

这个项目会接触本地微信数据、导出文件和飞书上传流程，因此我们尤其重视以下几类问题：

- 登录态、Cookie、会话文件泄露
- 明文持久化敏感数据
- 导出目录或临时目录中的隐私残留
- 上传结果误判导致的数据错误或误操作
- 不安全的默认配置、权限边界或历史泄露问题

## 请不要公开提交的内容

如果你发现的是安全问题，请不要直接在公开 Issue 或 PR 中贴出以下内容：

- Cookie、Token、登录二维码会话信息
- `auth_state`、`storage_state` 等浏览器登录态文件
- 真实微信数据库、解密后的数据库、用户隐私数据
- 可直接复用的账号凭据或访问地址

## 推荐的报告方式

如果问题涉及敏感信息，请优先通过仓库维护者可用的私下联系方式进行报告；如果暂时没有私下渠道，请先在公开渠道中只描述影响范围，不要公开细节与样本数据。

你可以按下面的结构提交安全报告：

1. 问题类型
2. 影响范围
3. 复现步骤
4. 预期风险
5. 建议修复方式
6. 是否已经包含敏感数据

## 修复与披露建议

- 在修复完成前，尽量避免公开完整利用细节
- 如问题涉及历史泄露，请同步评估是否需要轮换密钥、Cookie 或相关凭据
- 如问题涉及 Git 历史，请同步清理本地和远端历史

## 支持策略

当前优先关注最新代码和最新打包产物中的安全问题。历史版本如果没有继续维护，通常不会单独回补，但欢迎提交修复建议。
